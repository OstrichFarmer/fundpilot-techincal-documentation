RevenueCat & Entitlement Synchronization

Managing premium access requires a high-fidelity sync between the mobile client (Source of Truth for Purchase) and our Go Backend (Source of Truth for Background Jobs).

1. The Flutter Entitlements Flow

We utilize a centralized SubscriptionNotifier (StateNotifier) to manage the purchase lifecycle. The flow follows these strict steps:

// lib/features/subscription/presentation/providers/subscription_provider.dart

Future purchasePackage(Package package) async {
  state = true; // Set loading
  final result = await repository.purchasePackage(package);
  
  return result.fold(
    (failure) => false,
    (success) {
      if (success) {
        // Step 2: Trigger backend sync immediately
        ref.read(authProvider.notifier).updateUserPremiumStatus(true);
      }
      return success;
    }
  );
}

2. Transaction Verification & Backend Mirroring

When a purchase is completed, the mobile app hits our POST /api/subscriptions/sync endpoint. This ensures that features triggered from the server (like 9:00 AM AI Daily Summaries) know the user is premium without checking RevenueCat APIs on every job run.

  • Entitlement ID: We specifically check for the premium entitlement.
  • GORM Integration: The backend updates the users table, setting is_premium = true. This column is indexed for rapid checks during notification tickers.
  • Grace Period Management: RevenueCat webhooks (planned) or scheduled mobile-side checks handle subscription expirations to flip the database flag back to false.

Twelve Data: High-Density Batching

To provide real-time updates while respecting the **Twelve Data Grow Plan** (55 credits/min), we implemented a batching engine specialized for stock and crypto symbols.

Rate Limit Strategy: We budget 40 credits per minute for automated background refreshes, leaving 15 credits reserved for user-initiated searches and real-time quotes in the detail view.

1. Symbol Chunking Logic

Instead of single-request API calls, the PriceUpdater in Go gathers all unique symbols across the entire database and processes them in batches of 10.

// backend/internal/modules/marketdata/updater.go

batchSize := 10
for i := 0; i < len(symbols); i += batchSize {
    end := i + batchSize
    if end > len(symbols) {
        end = len(symbols)
    }
    
    batch := symbols[i:end]
    p.fetchPricesBatch(batch)
}

2. Complex API Response Parsing

Twelve Data returns a different JSON structure depending on whether a single symbol or multiple symbols are requested. Our Go implementation uses a dynamic decoder to handle both cases without error:

  • Multi-Symbol: Parses a nested map of quote objects.
  • Format Cleaning: Automates comma removal from string values (e.g., "$1,234.50" → 1234.50) to ensure mathematical precision in portfolio profit/loss calculations.
  • Error Isolation: One failed symbol in a batch does not prevent the update of the other 9 symbols.

AI-Powered Risk Analysis (Gemini Pro)

FundPilot doesn't just show numbers; it interprets them. We pass structured portfolio distributions to Google's Gemini Pro via the Go SDK.

Structured Prompting

We feed the AI three specific data points for every analysis:

  1. Asset Mix: The percentage breakdown of Stock, Crypto, and Manual assets.
  2. Profit/Loss Delta: Real-time absolute and percentage performance.
  3. Risk Indicators: Standard deviation and Sharpe ratio (calculated locally in the Risk module).

Automated Insights

  • Loss Alerts: Triggered immediately on the backend if an asset drops >2%. The AI generates a 2-sentence tactical recommendation (e.g., "Market volatility is affecting $AAPL; consider your long-term position").
  • Narrative Summaries: Every morning at 9:00 AM, the system summarizes the previous day's movements into a "Daily Wealth Report" narrative.
